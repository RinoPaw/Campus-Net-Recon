# 02-锐捷网关的幻术

## 2.1 教室里的“断网”
在宿舍调试好的静态 IP 方案，带到教室后失效了。连接 Wi-Fi 后，认证页面无法弹出，网络完全不通。我关闭了静态 IP，重新获取地址，发现网关变成了 `10.201.255.254`，`Prefix Length` 也从 15 变成了 16。

我尝试 Ping 了一下新网关，TTL 为 64；Ping 宿舍的电脑，TTL 为 63；Ping 宿舍的网关，TTL 依然是 64。这让我怀疑：这些网关 IP 背后，是不是同一台物理设备？

我用 `nmap` 扫了一遍：
```bash
nmap -sn 10.0-255.255.254
```
结果显示，`10.201.255.254` 到 `10.207.255.254` 共5个网关的延迟都在几毫秒量级。这证实了它们大概率是同一台核心交换机在不同 VLAN 接口的逻辑映射。 `10.202.255.254` 没响应，因为宿舍区域是特殊的`/15`大网段，网关为`10.203.255.254`

## 2.2 跨网段的静态 IP 实验
在宿舍时配置静态ip大部分时候都能成功，但当我尝试在教室手动配置静态 IP，一次都没成功。即使我手动把网关改成宿舍的 `10.203.255.254` 或者 `10.205.255.254`，网络依然无法连通。

使用 `traceroute` 追踪宿舍电脑的 IP 时，结果如下：
```text
1  10.201.255.254 (10.201.255.254)  2.691 ms
2  10.203.207.207 (10.203.207.207)  4.646 ms
```
无论我怎么修改 `Prefix Length` 或网关设置，数据包的第一跳永远指向当前环境的网关 `10.201.255.254`。这说明**路由表是由接入层的交换机/AC 下发的，终端的静态配置无法绕过 VLAN 隔离**。

## 2.3 AP 内部的二层互通
在教室再次进行 ARP 扫描，发现每秒 30 次的 Proxy ARP 依然存在。但在 `10.201.8.0` 到 `10.201.14.255` 范围内，我能发现大量真实的 MAC 地址；而一旦越过 `10.201.15.x`，真实设备就消失了。

推测我现在连接的物理范围是 `10.201.8.0/21`。

为了验证同 AP 内的互通性，我用 `iw dev wlan0 link` 确认两台手机连接的 BSSID（AP 的物理地址）完全相同。随后，我手动执行 `ip neigh add` 强制绑定了双方的 MAC 地址。

*   **结果**：互 Ping 时 TTL 确实变成了 64，实现了二层直连。
*   **非对称现象**：如果仅修改 A 手机的 ARP 表，A Ping B 时 TTL 为 64，但 B Ping A 时 TTL 仍为 63。

## 2.4 AP 间的隔离
随后，我尝试将两台手机连接到不同的 AP（BSSID 不同）。此时，无论我如何配置 ARP 缓存，Ping 的 TTL 始终固定在 63。

这说明：
1.  **同 AP 内部**：只要手动绕过 Proxy ARP，流量可以在二层交换，实现 TTL 64 的直连。
2.  **跨 AP 之间**：流量必须经过上层交换机进行三层转发，无论怎么操作，数据包都会被强制送入路由路径，TTL 必然衰减。
