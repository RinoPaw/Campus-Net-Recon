# 02-锐捷网关的幻术

## 2.1 教室里的“断网”
在宿舍调试好的静态 IP 方案，带到教室后失效了。连接 Wi-Fi 后，认证页面不弹出，网络不通。关闭静态 IP，重新获取地址，网关变成了 `10.201.255.254`，`Prefix Length` 也从 15 变成了 16。

Ping 了一下新网关，TTL 为 64；
Ping 宿舍电脑，TTL 为 63；
Ping 宿舍网关，TTL 依然是 64。
我怀疑这些网关 IP 背后是同一台物理设备。

`nmap` 扫描了一遍：
```bash
nmap -sn 10.0-255.255.254
```
```text
...
Nmap scan report for 10.201.255.254
Host is up (0.023s latency).
MAC Address: 28:D0:F5:74:30:0E (Ruijie Networks)
Nmap scan report for 10.203.255.254
Host is up (0.094s latency).
Nmap scan report for 10.205.255.254
Host is up (0.037s latency).
Nmap scan report for 10.206.255.254
Host is up (0.091s latency).
Nmap scan report for 10.207.255.254
Host is up (0.041s latency).
Nmap done: 256 IP addresses (5 hosts up) scanned in 7.59 seconds
```
结果显示，这几个网关的延迟都在几毫秒量级，大概率是同一台核心交换机在不同 VLAN 接口的逻辑映射。`10.202.255.254` 和 `10.204.255.254` 无响应，可能是因为宿舍区域属于特殊的 `/15` 网段，或者其他配置原因。推测校园 Wi-Fi 的范围是 `10.200.0.0/13`，这也解释了第一章中发现的 `10.200.1.5`。

## 2.2 跨网段的静态 IP 实验
在宿舍配置静态 IP 大多能成功，但在教室手动配置，一次都没成功。

但如果将网关改为宿舍的 `10.203.255.254`，或其他的网关，网络则不会受影响。

尝试追踪到宿舍的数据包：

```bash
traceroute -i 10.203.207.207
```

```text
1  10.201.255.254 (10.201.255.254)  2.691 ms
2  10.203.207.207 (10.203.207.207)  4.646 ms
```

```bash
ping -R -c 1 10.202.255.254
```

```text
64 bytes from 10.202.255.254: icmp_seq=1 ttl=63 time=31.6 ms
RR:     10.201.34.85
        10.203.255.254
        10.202.255.254
        10.202.255.254
        10.201.255.254
        10.201.34.85
```

无论怎么修改 `Prefix Length` 或网关，数据包的第一跳永远指向当前环境的网关 `10.201.255.254`。

## 2.3 AP 内部的二层互通
在教室再次进行 ARP 扫描，每秒 30 次的 Proxy ARP 依然存在。但在 `10.201.8.0` 到 `10.201.14.255` 范围内，能发现大量真实 MAC 地址。

我用 `iw dev wlan0 link` 确认两台手机连接的 BSSID 完全相同。随后，手动执行 `ip neigh add` 强制绑定双方 MAC 地址。

*   **结果**：互 Ping 时 TTL 变为 64，实现了二层直连。
```bash
ping -R -c 1 10.201.32.87
```
```text
64 bytes from 10.201.32.87: icmp_seq=1 ttl=64 time=257 ms
RR:     10.201.34.85
        10.201.32.87
        10.201.32.87
        10.201.255.254
        10.201.34.85
```

*   **非对称现象**：仅修改 A 手机 ARP 表时，A Ping B，TTL 为 64；但 B Ping A，TTL 仍为 63。
```bash
ping -R -c 1 10.201.34.85
```
```text
64 bytes from 10.201.32.87: icmp_seq=1 ttl=63 time=61.9 ms
RR:     10.201.34.85
        10.201.32.87
        10.201.32.87
        10.201.255.254
        10.201.34.85
```

## 2.4 AP 间的隔离
随后，将两台手机连接到不同的 AP（BSSID 不同）。此时，无论如何配置 ARP 缓存，Ping 的 TTL 始终固定在 63。不能下定论，因为有时候连到了不同 AP，被分配到了同一个 `/21` 网段，还是能扫到很多真实 MAC。
