# 01-奇怪的子网掩码

## 1.1 网管委之外的世界

起初我对校园网并没有太多兴趣。因为访问公网需办理新手机卡、绑定到账号，并每月缴纳 15 元网费。否则，连接 Wi-Fi 登录后，只能访问内网。而我已有两张卡，也不想额外付费。

我曾尝试不办卡，自行缴纳网费并绑定手机号。操作完成后账号随即无法登录，提示“您未绑定服务对应的运营商！”。我想改回原来的免费套餐，但提示一个月只能更改一次。于是等了一个月账号才恢复正常。再一看网费也扣除了。我觉得这是不地道的。

## 1.2 子网掩码 `255.254.0.0`

后来，杰哥给我分享了他的校园网账号。他日常流量充足，不常用校园网。我便开始使用他的账号。

一天，我在宿舍查看笔记本的网络属性，发现子网掩码是** `255.254.0.0`**。

我此前未见过子网掩码中出现 254。浅浅思考，子网掩码本质上是与 IP 地址进行“与运算”的 32 位二进制。那1 的中间是否可以有 0？我认为子网掩码的核心作用是将 IP 地址分组，连续性并不重要。但我尝试输入 `255.253.0.0`，Windows 报错。随后我在手机网络设置中发现 `Prefix Length` 属性，默认为 15。这表明子网掩码由一段连续的 1 和一段连续的 0 组成，且 1 在高位。这应该是一种规范。

## 1.3 局域网串流

我又查看了当前 IP：`10.202.40.xxx`，网关是 `10.203.255.254`。在 `255.254.0.0` 掩码下，它们确实处于同一网段。

我 Ping 了网关：
```bash
ping -c 4 10.203.255.254
```
```
--- 10.203.255.254 ping statistics ---
4 packets transmitted, 4 received, 0% packet loss, time 3004ms
rtt min/avg/max/mdev = 3.902/11.175/15.506/4.493 ms
```
我想到，如果手机和电脑都连接上校园网，能否实现局域网串流？我将手机也连接校园网，发现可以 Ping 通。于是我用Moonlight和Sunshine实现了延迟 30ms 以内的串流方案。

## 1.4 第一次“碰壁”

为方便使用，我尝试给电脑和手机设置静态 IP。电脑设置为 `10.203.xxx.xxx`，手机设置为 `10.202.xxx.xxx`。

但校园网限制同时登录 2 台设备。我不能霸占这两个登录名额，因为杰哥也分享了账号给其他人。我借用室友的校园网进行测试，发现连接校园网但不登录可以 Ping 通网关，但 Ping 其他设备会超时。登录校园网后，即使不绑定手机号也可以 Ping 通校内设备。

简单地说：

| Ping连通性 | 校域网 | 公网 |
| :---  | :--- | :--- |
| **不登录** | 限网关和登录页面 | 不可访问 |
| **登录但不开通套餐** | 可访问 | 不可访问 |
| **登录并开通套餐** | 可访问 | 可访问 |

这可以在一定程度上解决问题。我可以在电脑上登录室友的账号，手机上登录杰哥的账号。这样手机既能访问公网又能和电脑互联。但这样电脑就不能上网了。

一个思路是将电脑的网关直接设置为手机。我 `tracert` 了一下 8.8.8.8：
```powershell
tracert -d 8.8.8.8
```
```
Tracing route to 8.8.8.8 over a maximum of 30 hops
  1     3 ms     5 ms     4 ms  10.203.255.254
  2     1 ms     1 ms     1 ms  10.200.1.5
  3     *        *        *     Request timed out.
  ...
```
数据包根本没有经过手机，而是被直接发送到网关。问了AI ，提到了 **ARP 代理 (Proxy ARP)** 。

## 1.5 扫描：网关的响应

我用 `nmap` 扫描了该网段：
```bash
nmap -sn 10.202.40.0/24
```

扫描结果显示，大部分 IP 都指向同一个 MAC 地址：
```
(10.202.40.0-10 MAC Address)
28:D0:F5:74:30:0E (Ruijie Networks)
28:D0:F5:74:30:0E (Ruijie Networks)
28:D0:F5:74:30:0E (Ruijie Networks)
A2:35:B2:C9:90:13 (Unknown)
44:AF:28:F4:B9:E7 (Intel Corporate)
28:D0:F5:74:30:0E (Ruijie Networks)
28:D0:F5:74:30:0E (Ruijie Networks)
28:D0:F5:74:30:0E (Ruijie Networks)
28:D0:F5:74:30:0E (Ruijie Networks)
28:D0:F5:74:30:0E (Ruijie Networks)
```

这证实了 ARP 代理的存在。单纯的网关配置无法解决问题，而且在Android上配置转发规则也比较麻烦，最终我采用了手机上使用 `Clash Meta for Android` 开启 `Allow LAN` ，配合电脑端的 `Clash Verge Rev` ，实现了校内互联、公网访问且只占用一个名额的方案。
